# Billing

**Route:** `/billing`

> **Scenario:** It's the 20th of the month. You get an email saying you're at 85% of
> your report limit. You want to know when you'll hit the wall, whether it's worth
> upgrading now, and what yesterday's invoice looks like — without logging into Stripe.

Everything you need to manage your subscription is on this page: live usage with a
forecast, plan comparison, invoices, and in-console support if something goes wrong.

---

## Reading your usage card

Each project has a billing card. The key element is the **usage bar**:

```
Reports used: ████████░░░░  85%  (8,500 / 10,000)
```

Below the bar, the **forecast narrative** tells you what to expect:
> *"At this rate, you'll reach your limit on Jun 14."*

This is calculated from your rolling 30-day average. If you're growing fast, the date
moves sooner — upgrade before it hits to avoid queued (unprocessed) reports.

**What happens at 100%?** New reports are queued but not classified or fixed. They'll
process once the new billing period starts (or you upgrade). No reports are lost.

### Usage breakdown

| Metric | What it measures |
|--------|-----------------|
| **Subscription status** | `active`, `past_due` (update card), `canceled` |
| **Period end date** | When your current billing cycle resets |
| **Fix count** | How many fix-worker runs this month |
| **Classifier tokens** | LLM tokens used by the classifier pipeline |
| **LLM cost** | Estimated AI spend billed from your BYOK keys |

The **30-day sparkline** shows daily report volume — useful for spotting a recent spike
that's about to push you over the limit.

## Upgrading your plan

1. Click **Upgrade** (or **Change plan**) on the project card.
2. Browse the plan comparison table — it shows exactly which features each tier unlocks
   (SSO, data residency, higher limits, etc.).
3. Click **Select** on the plan you want.
4. You're redirected to **Stripe Checkout** — complete payment there.
5. The new plan activates immediately when you return to Mushi.

<Callout type="info">
If you don't see a plan comparison table, click **Show plans**. The table is hidden by
default once you're on a paid plan to reduce clutter.
</Callout>

## Managing payment and invoices

Click **Manage** to open the **Stripe Billing Portal** in a new tab. Use the portal for:
- Updating your credit card
- Viewing upcoming invoices
- Cancelling a subscription
- Downloading invoices as PDFs (also available directly from the Invoices table below)

The **Invoices table** on this page shows the last few invoices:

| Column | Description |
|--------|-------------|
| **Number** | Stripe invoice number |
| **Period** | Billing period covered |
| **Amount** | Charged (USD) |
| **Status** | `paid`, `open` (unpaid), `void` |
| **View** | Opens the hosted Stripe invoice page |
| **PDF** | Direct PDF download |

## In-console support tickets

<Callout type="info">
Support tickets are available on **Starter** plan and above. Hobby plan users are
directed to GitHub Discussions.
</Callout>

Click **Open ticket** to expand the support form. Don't leave Mushi to email support —
submit here and the ticket is linked to your project automatically.

**Filing a good support ticket:**
1. Select the **project** the issue affects.
2. Choose a **category** — "billing" for invoice questions, "bug" for product issues.
3. Write a clear **subject** (e.g. *"Fix-worker failing with 401 since key rotation"*).
4. Add detail in the **body** — include the run ID or report ID if relevant.
5. Click **Send ticket**.

The ticket appears in **Recent tickets** immediately. Click the row to open the detail
modal, where you'll see any admin reply. Click **Cancel ticket** if the issue resolves
itself before a reply arrives.

## Common tasks

### "I'm approaching my limit — should I upgrade now?"
1. Read the forecast narrative on your usage bar.
2. If it says you'll hit the limit before month-end: upgrade.
3. If you're at 85% with 2 days left: the new billing period will reset first — wait.

### "I was charged twice this month"
1. Open the **Invoices** table — check the Period column for each invoice.
2. If you see two invoices for the same period, click **Open ticket** → category: billing.

### "I need to cancel"
1. Click **Manage** → Stripe Billing Portal → Cancel subscription.
2. Your project continues until the period end date shown on the billing card.
3. Data is retained for 30 days after cancellation before automated deletion.
