# UX + design-system audit — 2026-04-19

> Live page-by-page audit of the mushi-mushi admin console (port 6464) with the
> glot.it production dogfood project active. The PDCA loop was kept hot during
> the sweep so every page rendered against real, fresh data instead of fixtures.
>
> Goal: turn three user pains into a fix list.
>
> 1. *"What do I do first?"* — first-action clarity
> 2. *"How does mushi-mushi map to my PDCA loop on glot.it?"* — narrative
> 3. *"The UI is unpolished and visuals don't earn their pixel-cost"* — design
>    system + data-viz tightening
>
> Companion screenshots live in `audit-before/01-…` (24 PNGs). The post-redesign
> sweep will land in `audit-after/`.

## 2. Cross-cutting findings (apply to most pages)

### 2.1 First-action clarity ("what do I do first?")

| symptom | evidence |
|---|---|
| Dashboard hero is a row of equal-weight KPI tiles → no single "do this next" | `audit-before/01-dashboard.png` |
| The PDCA cockpit is below the fold of the first viewport on a 1280-tall display once `<PageHelp/>` is open | same |
| `SetupChecklist` in banner mode is shown even when 4/5 steps are done — disproportionate vertical real-estate | same |
| Onboarding page ([`/onboarding`](apps/admin/src/pages/OnboardingPage.tsx)) opens with five identical wizard cards. No mention of glot.it or *why* I'm doing this | `audit-before/02-onboarding.png` |
| Sidebar lists 23 destinations flat — no progressive disclosure, no "this is the next stage" hint | `audit-before/01-dashboard.png` (left rail) |

**Fix path (Phase 2).**
- Add a `HeroIntro` strip pinned to the dashboard top: one sentence + one CTA
  (the "current focus" stage). PDCA cockpit moves *under* it as the supporting
  story.
- Onboarding gets a project-narrative strip so first-time users see exactly how
  glot.it (or whatever project is active) flows through Plan → Do → Check → Act.
- Sidebar gains progressive disclosure: collapse "Workspace" by default once a
  project has any reports, and add a small "← current focus" marker on the
  active stage row.
- Add a reusable `<PdcaContextHint stage="plan" />` primitive every page can
  drop in its header so the user always knows where they are in the loop.

### 2.2 PDCA narrative ("how does this fit my loop?")

| symptom | evidence |
|---|---|
| `PdcaCockpit` stage tiles are visually identical-weight; "current focus" only earns a 1px ring | `audit-before/01-dashboard.png` |
| The bottleneck callout sits *below* the tiles — easy to miss | same |
| No connecting line / arrow between stages on small screens (only at lg+) | same |
| Reports table groups by nothing, so duplicate fingerprints bury the unique ones — masks the dedup story | `audit-before/03-reports.png` |
| Graph storyboard labels show categories ("ui", "auth") instead of the actual component / file names from the knowledge graph | `audit-before/04-graph.png` |
| Fixes timeline doesn't render the PDCA "receipt" (Plan source → Do PR → Check verdict → Act ship) | `audit-before/07-fixes.png` |

**Fix path (Phases 2 + 4).**
- Rework `PdcaCockpit`: the current-focus stage gets a 2× larger number, its own
  brand-tinted background, and the bottleneck callout is *inside* its tile.
- Add a `<FixGitGraph/>` mini view to each fix row showing the four-stage
  receipt with timestamps.
- Re-label `<GraphStoryboard/>` chapters with real component names + write 1-line
  story copy ("3 bugs trace back to `<CommandPalette/>` in v2.4").

### 2.3 Design-system polish

| symptom | evidence |
|---|---|
| `Btn` only ships hover; no explicit `:active` press scale, no loading slot, no `:focus-visible` ring (it inherits the global one but it's faint) | code: `apps/admin/src/components/ui.tsx:567` |
| `Toggle` thumb has no shadow/ring change on focus | same file `:659` |
| `Input` & `SelectField` border ring is `brand/40` — only ~40% opacity, low contrast on dark surface | same file `:591` and `:612` |
| Many pages use raw Tailwind colour classes (e.g. `bg-zinc-700`, `text-amber-400`) instead of the `--color-*` semantic tokens | `rg "bg-zinc-|text-amber-" apps/admin/src` returns 30+ hits |
| Card vertical rhythm is inconsistent — some pages use `mb-3`, others `mb-4`, others bare `space-y-2` | grep across `apps/admin/src/pages/*.tsx` |
| Empty states lack a recommended action and a tone (everything is the same neutral grey card) | `EmptyState` in `ui.tsx:707` |

**Fix path (Phase 3).**
- Sweep token compliance across `apps/admin/src` (one PR; mechanical).
- Extend `Btn`, `Input`, `Toggle`, `Select` with the full state matrix:
  default · hover · `:focus-visible` · `:active` · disabled · loading · error.
- Standardise vertical rhythm on `space-y-4` for page sections, `space-y-2.5`
  for in-card stacks; codify with a `<PageStack/>` wrapper.
- Layer in personality: keep the brand glow on the active nav rail, add a soft
  pulse to the bottleneck dot, ease all entrance transitions with the existing
  `--ease-out-expo`.

### 2.4 Data-viz that doesn't earn its pixels

| symptom | evidence |
|---|---|
| `KpiTile` shows a number + sublabel but no *meaning copy* — what does "Reports (14d)" *mean* for me? | `apps/admin/src/components/charts.tsx:46` |
| Severity stack on the dashboard has no per-day total label and no hover breakdown | same file `:198` |
| Sparkline on triage backlog doesn't show severity composition — just a flat line | code review |
| Reports table columns repeat the same status text and never collapse duplicates | `audit-before/03-reports.png` |
| LLM call list on `/health` is a long flat table — no grouping by function, no cost summary per function | `audit-before/10-health.png` |

**Fix path (Phase 4).**
- `KpiTile` gains an optional `meaning` slot ("up means more friction reaching
  users — investigate sources").
- `SeverityStackedBars` learns hover breakdown + per-day total.
- `ReportsTable` learns "group by fingerprint" (collapse duplicates with a
  count badge and "blast radius" column = unique users hit / 7d).
- `/health` gets a per-function cost + p95 latency strip above the call list.

## 3. Page-by-page findings

Severity: **P0** = blocks core understanding, **P1** = high-impact polish,
**P2** = nice-to-have.

### Plan stage

#### `/` Dashboard — `audit-before/01-dashboard.png`
- **P0** No hero "first action" — user has to read 4 KPI tiles + 4 PDCA tiles + a checklist before knowing what to do.
- **P0** PDCA cockpit "current focus" ring is too subtle (1px, low-contrast).
- **P1** `SetupChecklist` banner stays even at 4/5 done — should collapse to a single-line "All set ✓" once mostly complete.
- **P1** No live "Last report N seconds ago" pulse to communicate the loop is hot.
- **P2** `QuickFiltersCard` at the bottom is a dumping ground; either promote to header or remove.

#### `/onboarding` — `audit-before/02-onboarding.png`
- **P0** No project-narrative strip — the page reads like generic SaaS docs, not "here's how mushi will help you on glot.it specifically".
- **P0** Wizard cards have no concept of "current step" — all five look the same.
- **P1** "Send test report" lives at the bottom; the most important interactive proof should be at the top.

#### `/reports` — `audit-before/03-reports.png`
- **P0** Duplicate reports (same fingerprint) clutter the list — no group-collapse.
- **P0** Status pills don't visually convey stage progression (new → classified → fixing → fixed).
- **P1** No "blast radius" column (unique users hit) — hides mushi's dedup value.
- **P1** KPI strip on top doesn't include severity breakdown — only counts.
- **P2** Page header has no PDCA stage chip ("Plan").

#### `/graph` — `audit-before/04-graph.png`
- **P0** Storyboard chapters use category names (`ui`, `auth`) instead of actual component / file paths from the graph — useless for a real codebase.
- **P1** No narrative copy ("3 bugs trace back to `<CommandPalette/>` in v2.4") — graph is just dots.
- **P2** Edge weights aren't legend-explained.

#### `/anti-gaming` — `audit-before/05-anti-gaming.png`
- **P1** Page never explains what "Anti-gaming" means in the loop. Add a one-line PDCA hint and a link to the SDK docs.
- **P2** Counters present but no "today vs 7d" delta.

#### `/queue` — `audit-before/06-queue.png`
- **P0** "Recover stranded" button is small + unlabeled — given it's a self-healing power tool, deserves prominence + a tooltip with the time threshold.
- **P1** Pending items don't show how long they've been pending (`waiting Xm`).

### Do stage

#### `/fixes` — `audit-before/07-fixes.png`
- **P0** No PDCA "receipt" per fix (Plan source → Do PR → Check verdict → Act ship). Today users see a fix card with no context of which report it solved.
- **P1** Status filter is hidden in a tiny `<select>`; promote to a segmented control.
- **P2** Open PR rows don't link to the GitHub PR directly (only via overflow).

#### `/prompt-lab` — `audit-before/08-prompt-lab.png`
- **P1** Compare view has no diff view between prompt versions; only side-by-side text.
- **P2** Token-cost comparison is missing — would clinch the value.

### Check stage

#### `/judge` — `audit-before/09-judge.png`
- **P1** "Run judge now" is a small button at top-right; for a power surface, give it a primary CTA.
- **P2** Disagreement column has no narrative — show the 1-line judge note inline.

#### `/health` — `audit-before/10-health.png`
- **P1** LLM call list is flat — group by `function_name` with sub-total cost / p95 latency.
- **P1** No "last failure" chip per function.

#### `/intelligence` — `audit-before/11-intelligence.png`
- **P1** Page reads as a stat dump; no story ("This week, glot.it shipped 3 fixes that cut user-felt errors by 40%").

#### `/research` — `audit-before/12-research.png`
- **P2** Search history has no filter by query type / date.

### Act stage

#### `/integrations` — `audit-before/13-integrations.png`
- **P1** Integration cards don't show "what you can do once connected" — only generic descriptions.
- **P2** No quick re-test action per integration.

#### `/marketplace` — `audit-before/14-marketplace.png`

#### `/notifications` — `audit-before/15-notifications.png`
- **P2** Channel test buttons (Slack/email) don't show last test status.

### Workspace

#### `/settings` — `audit-before/17-settings.png`

#### `/sso` — `audit-before/18-sso.png` — OK as-is, P2 only on visual rhythm.
#### `/billing` — `audit-before/19-billing.png` — usage bars are correct; **P1** add forecast band ("at current rate, you'll hit limit on …").
#### `/audit` — `audit-before/20-audit.png` — **P2** no filter by actor type.
#### `/compliance` — `audit-before/21-compliance.png` — **P2** add a single PDF export.
#### `/storage` — `audit-before/22-storage.png` — **P2** no per-project breakdown.
#### `/query` — `audit-before/23-query.png` — **P1** no saved-queries list, no SQL syntax hint card.

### Report detail — `audit-before/24-report-detail.png`
- **P0** No "PDCA receipt" header (which stage am I in for *this* report?).
- **P1** Screenshot is rendered small even though it's the most evidence-rich asset; promote to a hero strip with `<ImageZoom/>`.
- **P2** Recommended action is generic; should reference category-specific advice.
